Vim Basics -
Learn the Most Important Vim Commands in Just a Few Minutes

Vim is one of the most popular text editors for professionals and hobbyists who write code (according to the Stack Overflow Developer Survey; sys admins seem to LOVE Vim) and thanks to the fact that Vim can easily be modified and extended, it is not just a simple editor with a command line interface but a fully-featured development environment that provides syntax highlighting, auto-completion, a directory tree and many more features.

This sounds great and software (or web) developers who use editors like Sublime or Atom will certainly appreciate the multitude of available plugins. However, the 'Vim Language' itself is extremely powerful. That said, it is certainly not easy for beginners or people who want to transition from one of the above mentioned editors to Vim, to get used to 'The Vim Way' of doing things.

I wrote this post to summarize all the commands in Vim that I use on a daily basis and the resources I found (and find) most useful when learning Vim (and this learning process does probably never end). Hopefully, putting all these things in one place will provide a starting place for one or two people who want to get started with Vim but were overwhelmed in the past by all the new things one needs to learn before being productive with Vim.

The first version of Vim dates back to the year 1991 when v1.14 was publically released. Vim is actually an acronym and stands for Vi IMproved. Vims predecessor Vi was developed as a text editor program for Unix and originally released in 1978. Thus, the source code for Vi/Vim is around for more than 4 decades. Vim was constantly extended since the earliest days and nowadays it comes with an entire universe of plugins.

Vim as a text editor is based on the idea that you will usually spend more time editing and changing text/code than writing it from scratch. Many of the commands in Vim are optimized with this 'philosophy' in mind. To make modifying text as fast, flexible and intuitive as possible, Vim provides its user with a complete language; meaning, Vim has nouns, verbs, and adverbs as most languages.


Nouns / Text Objects

Text objects can be interpreted as Vim's nouns. Text objects are single words (keyword w), sentences (keyword s), whole paragraphs (keyword p), tags (keyword t), and many more. The can be used to specify which part of your text/document you want to modify - or they can be used as motions. That means that typing for example w in normal mode (see below) lets you jump to the next word. In a similar way, combining w with an number lets you jump the indicated number of words.


Verbs / Actions

Actions on text (like deleting or copying) can be interpreted as Vim's verbs. Most of these verbs are relatively intuitive, for example deleting is done with keyword d and changing text with keyword c. You can already see that the combination of nouns and verbs (e.g. dw for deleting a word) can be extremely powerful. Other actions allow you to save your file (w for write) and quite (keyword q).


Adverbs / Modifiers

Adverbs can be used to fine-tune verbs (again, pretty similar to natural languages). If you wanted to delete text inside parentheses, you could use the adverb i and searching/finding is done with with the keyword f. You will usually be able to combine an adverb, verb, and noun to perform a desired operation on your file with just 3 key strokes. That makes working with Vim very fast (as soon as you know a basic set of keywords).


Modes in Vim

All these keywords in Vim make it impossible to have just one mode of working with your file. There must be at least one writing mode (called insert mode) and one mode to move around the file (called normal mode according to Vim's philosphy). In fact, Vim has more than half a dozen different modes. But to get comfortable working with Vim and be productive, you might want to focus on the 3 most important modes first. In addition to the two modes outlined above, you might want to highlight parts of your text from time to time. That can be done in visual mode.
